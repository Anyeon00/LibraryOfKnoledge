그래프표현 2가지방법
-인접행렬	: 2차원배열사용
-인접리스트	: 연결리스트 사용

인접행렬 // 행과 열이 개수 = 정점의 개수
: if (간선 i,j 가 그래프에 존재) -> M[i][j] = 1 // M[j][i] 도  = 1 (대칭) 1에서0으로 이어지는거나 0에서 1로 이어지는거나_방향이없는경우
else			-> M[i][j] = 0

인접리스트
정점이 0, 1, 2, 3이 있고, 간선이 01, 02, 03, 12, 23인 경우
:
0 -> 1 ->2 ->3 ->null
1 -> 0 -> 2 -> null
2 -> 0 -> 1 -> 3 ->null
3 -> 0 -> 2 -> null

그래프탐색 : 깊이우선탐색, 너비우선탐색
-깊이우선탐색 : 인접한 정점을 방문하고 방문한 정점 표시후 방문하지 않은 인접정점을 방문 반복(재귀), 
인접한 곳을 모두 방문햇는데 방문하지 않은곳이 잇으면: 그전에 방문한 곳으로 돌아가기 반복
그러다가 제일 처음 시작한 지점으로 돌아가면 끝 _전위순회와 같음
-> 인접행렬로 구현된경우 코드
-> 인접리스트로 구현된경우 코드

신장트리
: 한 그래프내의 모든 정점을 포함하고 사이클이 없는
용도
: 최소 개수의 간선을 사용하여 통신 네트워크를 구축하고 싶은 경우
찾는방법
: 깊이우선탐색(전위순회)를 하면서 지나간 부분을 간선으로 표시

-너비우선탐색 : 인접한 정점들을 방문후, 그 다음으로 인접한 정점들을 방문 반복_큐를 사용하여 구현
방문한 곳을 표시한후 큐에 push, 인접한 곳을 모두 push햇으면 다시 pop하여 나온 정점에 대해서 다시 반복


경로
단순경로: 모든 정점들이 다른 경우
경로의 길이: 간선의 개수
최단경로를 찾을떄: 너비우선탐색사용
